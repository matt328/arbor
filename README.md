# Arbor

Arbor is a hobby game engine which doesn't do much right now other than (mostly) initialize Vulkan. It is a rewrite of the Triton engine, and its goals are mostly the same as Triton's but with some of the more prototype concepts removed. Most of the differences center around using vulkan's c api directly and creating a handful of thin raii style wrappers around vulkan's handles instead of relying on autogenerated wrappers from vulkan_raii.hpp, and removing the dependency on boost::di in favor of proper architecture instead of the DI soup that evolved in Triton.

Goals of the engine include:

- Potentially infinite world size combining procedural generation with hand crafted areas
- Realtime deformable terrain
- Realtime Global Illumination

Technical Details:

- Multithreaded background asset loading and uploading via transfer queue
- Gameplay fully decoupled from rendering with separate threads for each
- Fully bindless gpu driven rendering.
- Vertex pulling
